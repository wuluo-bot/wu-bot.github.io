虽然第二种写法看着复杂，但是，在今后的组件开发中，必须使用函数式的写法，而且不能使用箭头函数，因为箭头函数的 this 不是 Vue 实例对象，而是 Window。

总结

data 与 el 的 2 种写法

  1. el 有 2 种写法
    (1).new Vuel 时候配置 el 属性。
    (2).先创建 Vue 实例，随后再通过 vm.$mount(‘#root’)指定 el 的值。
  2. data 有 2 种写法
    (1).对象式
    (2).函数式
    如何选择：目前哪种写法都可以，以后学习到组件时，data 必须使用函数式，否则会报错。
  3. 一个重要的原则：
    由 Vue 管理的函数，一定不要写箭头函数,一旦写了箭头，this 就不再是 Vue 实例了。
